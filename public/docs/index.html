Documentation

BitBabit: Variant Images lets merchants assign multiple images per product variant, store them in Shopify (custom.variant_images), and display them on the product page with a theme app block or custom theme code.

Pricing

Limits apply when saving in the embedded Admin app. The storefront reads metafields directly from Shopify — see Privacy & data for how billing relates to the theme.

Free
$0 / month
  • Up to 6 images per variant
  • Up to 5 products with variant images
  • Theme app block & Liquid access
Pro
$9.99 / month
  • Unlimited images per variant
  • Unlimited products
  • Copy to another product (same variant count)
  • 7-day trial — confirm in app Pricing (subject to Partners listing)

Upgrade: In the app, open Pricing in the header and subscribe through Shopify App Store billing. Custom/private apps may not support in-app billing — your developer can use server configuration for Pro access where applicable.

Requirements

  1. A Shopify store with products that have variants.
  2. Metafield definition (recommended before heavy use):
    • Settings → Custom data → Variants
    • Namespace custom, key variant_images
    • Type: List of files (images)
    • Enable Storefront access so Liquid can read values on the Online Store.
  3. The app installed from the Shopify App Store or your install link.

Merchant workflow

  1. Open Apps → BitBabit: Variant Images (or your listing name).
  2. Select a product, then assign images per variant (reorder, upload, or pick from Shopify Files).
  3. Save each variant as needed.

Until images are saved, the storefront has nothing extra to show for that variant.

Storefront (PDP)

Two supported integration paths. Use tabs below to compare.

Recommended. No custom CSS selectors required: the block renders its own gallery (hero + thumbnails) where you place it.

  1. Online Store → Themes → Customize
  2. Open the Product template
  3. Add block → Apps → Variant image gallery
  4. Place the block in the main product area (e.g. beside the buy box)
  5. To avoid duplicate galleries, hide or remove the theme’s default product media block if you only want these images

Behavior uses common variant events (variant:changed, picker name="id") and ?variant= links when applicable. You can customize the empty state message in block settings.

For fully custom themes, integrate with a snippet that coordinates native media and metafield-driven UI (e.g. window.eventBus, section IDs, [data-product-media-wrapper]).

See the repository companion: docs/examples/variant-images-metafield-theme-integration.md for a full step-by-step reference (events, shell pattern, quick view).

Liquid quick reference

Read images for the selected variant on the PDP:

{% assign variant_images = product.selected_or_first_available_variant.metafields.custom.variant_images.value %}
{% if variant_images and variant_images.size > 0 %}
  {% for image in variant_images %}
    <img src="{{ image | image_url: width: 1200 }}" alt="{{ image.alt | default: product.title | escape }}" loading="lazy">
  {% endfor %}
{% endif %}

Use variant.metafields.custom.variant_images inside {% for variant in product.variants %} when building JSON for JavaScript.

Deploy & updates

  • After a new release: run shopify app deploy (or your CI pipeline).
  • Merchants may need to accept updated permissions or app versions when prompted.

Privacy, billing & data

Data Where it lives
Variant image file references Shopify variant metafield custom.variant_images
OAuth sessions Your app database (e.g. Prisma Session)
Free-tier product counts Your app database (usage tracking)
Subscription state Shopify Billing API (when applicable)

The Online Store reads metafields in Liquid; it does not call your app server on each page view. After uninstall, sessions are removed from your DB; variant metafields are not automatically deleted (merchants keep catalog data in Shopify).

For App Store privacy policy inputs and compliance webhooks, maintain internal docs aligned with Billing API and ShopUsage storage as described in your legal pages.

Troubleshooting

Check metafield Storefront access, that the variant has saved images in the app, and that the theme block is added or your snippet is integrated.
Confirm assignments in the app for that variant; hard-refresh the product page. Option A works on most OS 2.0 themes; custom themes may need Option B patterns.
That variant has no files in the metafield — assign images in the app and save.
Upgrade under Pricing in the app for Pro, or reduce the number of images per variant / products using variant images on Free.
